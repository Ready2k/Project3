## Documentation Standards

### File Organization
- **User-facing guides**: Keep in `docs/` directory
- **Core system docs**: Keep in root directory for visibility
- **Implementation summaries**: Archive in `legacy/docs/implementation-summaries/`
- **Historical docs**: Archive in `legacy/docs/`

### Naming Conventions
- Use descriptive, uppercase names for major guides (e.g., `AGENTIC_AI_GUIDE.md`)
- Use lowercase for technical documentation (e.g., `enhanced_patterns.md`)
- Include version or date in implementation summaries when archived

### Content Guidelines
- Keep documentation current and accurate
- Include practical examples and code snippets
- Provide clear step-by-step instructions
- Link to related documentation
- Update documentation when features change

## Contributing to Documentation

When adding or updating documentation:

1. **Determine the right location** based on the audience and purpose
2. **Follow naming conventions** and content guidelines
3. **Update this index** when adding new documentation
4. **Archive outdated docs** to legacy directory instead of deleting
5. **Cross-reference** related documentation with links

## Getting Help

For questions about the documentation:
